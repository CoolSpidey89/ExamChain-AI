# ExamChain — Secure, Fair & Intelligent Examinations

---

## 🧠 What is ExamChain?

ExamChain is a full-stack examination platform that solves three core problems with traditional MCQ-based exams:

- **Cheating** — Every student receives a uniquely assigned question variant, generated by AI from a shared concept pool. Copying answers between students is structurally useless.
- **Unfairness** — Scores are normalized into a true percentile rank, so performance is judged relative to peers and weighted by question difficulty — not just raw correct counts.
- **Integrity** — Once an exam is locked, no question can be edited or added. A tamper-detection hash chain runs quietly in the backend as an additional integrity layer.

---

## ✨ Features

### 👨‍🏫 For Teachers
- Register/login with full email OTP verification
- Create multiple exams, each with its own shareable 6-character admit code and configurable time limit
- Add questions with a concept tag and difficulty level — AI (via OpenRouter) generates 5 unique variants per question
- **Review and edit AI-generated variants** before they're saved — fix wording or correct answers before they go live
- Lock an exam to freeze its questions permanently
- View a full results dashboard: leaderboard, concept-wise weakness analytics, and a one-click release/hide toggle for student-facing scores
- Forgot password flow via OTP

### 👨‍🎓 For Students
- Register/login with college email, exam roll number, and full OTP verification
- Join any exam using a 6-character admit code
- Take the exam with a live countdown timer (server-enforced — refreshing doesn't reset the clock)
- Question navigation dots to jump between questions and track progress
- Auto-submit when time runs out
- See only a confirmation message immediately after submitting — no score is shown until released
- Once a teacher releases scores: view personal correct-answer count and percentile rank — never other students' data

### 📊 Scoring & Fairness
- Each question carries a difficulty weight (1–5), set by the teacher
- Raw performance is calculated as a difficulty-weighted score
- Final rank is a **true percentile** — the top scorer in any exam always lands at exactly the 100th percentile, with ties handled correctly
- A server-side integrity check refuses to serve results if the percentile math ever produces an invalid output, rather than silently shipping wrong numbers

## 🔐 Security Model

- **Authentication:** JWT-based, with bcrypt password hashing and OTP email verification gating both registration and login
- **Role separation:** Teacher and student accounts are strictly separated — the same email cannot register as both
- **Exam locking:** Once locked, no question can be edited, added, or removed via the application — enforced server-side regardless of frontend state
- **Tamper-detection hash chain:** Each question block is hashed together with the hash of the previous block (SHA-256), forming an append-only chain. This is verifiable via an internal API endpoint, separate from the teacher-facing UI, to prove no question content was altered after creation.
- **Variant assignment:** Variants are distributed to avoid repeats across students wherever possible, falling back to randomized reuse only once every variant has been issued at least once.
- **Server-enforced timing:** Exam start time is recorded server-side on first load; the deadline cannot be extended by refreshing or replaying requests.

## 🎨 Design System

ExamChain's visual identity is grounded in the language of physical exam stationery — admit cards, sealed answer sheets, official stamps — rendered through a modern glassmorphic UI with ambient animated backgrounds.

- **Palette:** ink black background, warm paper cream surfaces, brass gold (teacher/primary), muted green (student/success), oxblood red (alerts)
- **Typography:** Fraunces (serif, headings — gives an "official document" feel), Inter (UI text), JetBrains Mono (codes, timers, scores)
- **Signature elements:** the exam code rendered as a perforated admit-card ticket, a rotated "Sealed" stamp badge on locked exams, lettered-circle answer options, and a boxed OTP-style code entry for joining exams

## 🎯 Demo Flow

1. **Teacher** registers, verifies email via OTP, creates an exam with a duration
2. **Teacher** adds a question → AI generates 5 variants → teacher reviews/edits → confirms
3. **Teacher** locks the exam, gets a shareable 6-character code
4. **Student** registers, verifies email, joins the exam using the code
5. **Student** takes the exam with a live timer and submits — sees only a confirmation, no score
6. **Teacher** opens Results, sees the leaderboard and concept-wise weakness breakdown, releases scores
7. **Student** returns to their results page and sees their correct-answer count and percentile

## 🏆 Origin

This project began as a submission for the **Examinations** theme at FAR AWAY 2026, built around three pillars:

- ✅ **Security** — unique variants per student, locked exams, tamper-detection hashing
- ✅ **Fairness** — difficulty-weighted, percentile-normalized scoring
- ✅ **Intelligence** — LLM-powered question variant generation and concept-level analytics

It has since moved from hackathon prototype toward a real, deployable product with proper authentication, email verification, and a dedicated visual identity.
